# design-sync NOTES — Ontology Studio Design System

Repo-specific gotchas for future re-syncs. Read before running.

## Shape: off-envelope (Next.js app, not a library)

This package (`ontology`) is a Next.js app — no `dist/`, no library exports, no Storybook.
Handled via **synth-entry-by-barrel**:

- **Entry**: `.design-sync/entry.tsx` — hand-written barrel that `export *`s the scoped
  `src/components/ui/*` files (+ `confirm-card`). Passed to the converter via `--entry`.
  This is what makes the bundle contain only the DS surface (all compound sub-parts included)
  instead of the whole app. Add/remove a component here AND in `componentSrcMap`.
- **Component list**: `componentSrcMap` in config.json enumerates the ~30 card components
  (one primary per file) and pins each src path. Sub-parts (CardHeader, SelectTrigger, …)
  ride in via `export *` but aren't cards.
- `cfg.tsconfig: ./tsconfig.json` — needed so esbuild resolves the `@/*` → `./src/*` alias.

## CSS: compiled Tailwind v4 (the key to styled output)

Tailwind **v4.3.1** (CSS-first: `@import "tailwindcss"`, `@config`, `@plugin`, `tw-animate-css`).
Components are styled entirely with utility classes + CSS-var tokens, so the bundle needs a
**real compiled stylesheet**, not the `@tailwind` source.

- `cfg.buildCmd` compiles it: `node node_modules/@tailwindcss/cli/dist/index.mjs -i ./src/app/globals.css -o ./.design-sync/compiled.css` (~150KB, real utilities + `:root` token vars).
- `cfg.cssEntry: ./.design-sync/compiled.css` (gitignored — regenerated by buildCmd every run).
- **Re-run buildCmd before the converter** whenever component class usage or globals.css changes,
  else new utility classes won't be in the compiled CSS.

## Build / validate commands (from pkg dir)

```
node node_modules/@tailwindcss/cli/dist/index.mjs -i ./src/app/globals.css -o ./.design-sync/compiled.css
node .ds-sync/package-build.mjs --config .design-sync/config.json --node-modules ./node_modules --entry ./.design-sync/entry.tsx --out ./ds-bundle
node .ds-sync/package-validate.mjs ./ds-bundle
```

- `--node-modules ./node_modules` (pkg's own) — react@19 + all radix/cva/lucide resolve there.
- playwright **1.58.2** (repo pin) → chromium build **1208**, already in the local ms-playwright
  cache. Render check runs with no install.

## Fonts (known gap, acceptable)

Brand fonts (Pretendard / Outfit / JetBrains Mono) are injected at runtime by `next/font` as CSS
vars (`--font-pretendard`, …). Those vars are undefined in the bundle, so previews fall back to
system sans. No `[FONT_MISSING]` fires (no `@font-face` to satisfy). Text still renders, just not
in the brand face. To close this later: ship the woff2s + define the vars in a small `@font-face`
CSS and add via `cfg.extraFonts`, or set the vars in the compiled CSS.

## Known render warns (triaged — not new on re-sync)

- `[RENDER_BLANK]` on Avatar, Badge, Card, Checkbox floor cards — tiny/empty bare-component
  floor renders. Resolved once their `.design-sync/previews/*.tsx` are authored.
- tokens: 3 referenced-but-undefined CSS custom properties (below threshold, non-blocking).

## Preview composition gotchas (per-component)

All 30 previews render + grade good. Non-obvious composition choices that a re-author must keep:

- **Portals capture fine open** — Dialog/AlertDialog/Sheet/Popover/DropdownMenu/Select/Tooltip render their open state via `open`/`defaultOpen` (Tooltip needs `TooltipProvider`); the headless capture screenshots the page so body-portaled content IS captured. **No `cardMode` override was needed.** (If the DS-pane grid ever shows overflow, add `cfg.overrides.<Name>.cardMode`.)
- **ContextMenu** — pointer-triggered; radix has no static `open` and `forceMount` content animates to opacity-0 when closed. Preview shows the resting trigger surface only (graded good with a note). Open menu can't render statically.
- **Toaster** — live `<Toaster/>` is empty when idle and `ToastViewport` is `position:fixed` (off-card). Preview composes static `ToastProvider` + `Toast open` primitives with inline `style={{position:"static"}}` on the viewport + `duration={Infinity}`.
- **Sidebar** — `collapsible="none"` (bypass the `hidden md:block` desktop branch that blanks in a narrow card) + `SidebarProvider style={{minHeight:0}}` (neutralize `min-h-svh`) + fixed height.
- **Form** — `Form` IS `FormProvider`; each cell is a real component calling `useForm({defaultValues})` (react-hook-form bundles from node_modules); error state via literal `<FormMessage>` child.
- **Command / ScrollArea** — `h-full w-full`; need an explicit fixed-size bordered wrapper or they collapse.
- **FileUpload** — only required prop is `onFileChange` (no-op ok); body via `children`.
- **ConfirmCard/ConfirmBadge** — `VerdictKind` has 8 values in two tone families (dedup: reuse/relate/possible_duplicate/new; status: extend/fork/pass/block); `confidence` is raw 0–1 → qualitative band (omit for label-only); `attention`/`applied` booleans drive card framing.

## Re-sync risks

- `compiled.css` is derived + gitignored — always run buildCmd first (deterministic).
- Barrel + componentSrcMap must stay in lockstep; a component added to one but not the other
  either isn't bundled or isn't a card.
- Font gap above is unresolved by design — revisit if brand-font fidelity is needed.
