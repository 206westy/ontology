## 0. 개요

### 제품 비전
"도메인 전문가가 코드/쿼리 없이 온톨로지를 구축하고, 즉시 활용할 수 있는 그래프 편집 스튜디오"

### v3 전략 방향
**"적절한 표현력 + 최고의 접근성"**
- Protégé/TopBraid의 OWL 완전성 경쟁에 참여하지 않음
- LLM을 통한 접근성 우위를 극대화
- "만들고 끝"이 아닌 "만들고 바로 활용"하는 워크플로우 완성

### v3 핵심 목표
1. 온톨로지 품질 보장 메커니즘 도입 (제약조건, 유효성 검사, 중복 탐지)
2. LLM 활용 고도화 (AI 탭 활성화, 대량 파싱, 자연어 쿼리)
3. 사용자 경험 전면 개선 (온보딩, 입력 플로우, 커밋/푸시 단순화)
4. 디자인 시스템 정비 및 UI 품질 향상
5. 기술 기반 현대화 (의존성 업그레이드, AI SDK 도입)

## 2. 온톨로지 모델링 강화

> **담당**: 온톨로지 전문가 제안 기반

### 2.1 제약 조건(Constraints) 관리 시스템 — Must-have

**현재 한계**: 제약 조건 없는 온톨로지는 "구조화된 메모장"에 불과. 도메인 규칙을 시스템이 강제하지 못함.

**구현 범위**:
- **카디널리티 제약**: 관계별 min/max 설정 (UI: 숫자 입력)
- **도메인/레인지 강제**: 관계 생성 시 허용된 소스/타겟 클래스만 선택 가능
- **Disjoint 선언**: 클래스 쌍/그룹 간 배타 관계 설정
- **프로퍼티 값 제약**: range(min/max), pattern(정규식), enum 확장
- **자연어 → 제약 변환**: LLM이 사용자 입력을 구조화된 제약으로 해석

**DB 변경**: `constraints` 테이블 신규 + `edges` 테이블에 카디널리티 컬럼 추가

**사용자 시나리오**:
1. Equipment 클래스의 `located_at` 관계 선택 → "최소 1, 최대 1" 카디널리티 설정
2. Equipment 인스턴스 생성 시 Site 관계 없으면 CommitBar에 경고 표시
3. "DryAsher와 WetStation은 배타적" 자연어 입력 → LLM이 disjoint 제약 변환

### 2.2 스키마 유효성 검사 — Must-have

**검증 규칙 3단계**:
- **Error** (차단): 순환 is-a, 필수 프로퍼티 누락, 카디널리티 위반
- **Warning** (경고): 고아 노드, 빈 클래스, 프로퍼티만 있고 값 없는 인스턴스
- **Info** (참고): 유사 이름 중복 가능성, 미사용 관계 타입

**UI**: Toolbar "검증" 버튼 + 문제 목록 패널 (severity별) + 클릭 시 해당 노드 이동
**자동화**: 커밋/Neo4j 푸시 시 자동 검증, Error 있으면 차단

### 2.3 중복 탐지 및 병합 (Entity Resolution) — Must-have

**동작 방식**:
- LLM 프리뷰 단계에서 기존 엔티티와 유사도 비교 → "기존 'DryAsher'와 동일한가요?" 배지
- 사용자 선택: "병합" (기존 엔티티에 정보 추가) / "별개" (새 엔티티 생성)
- 주기적 "중복 검사" 실행 가능 (편집 거리 + 임베딩 유사도)

### 2.4 프로퍼티 상속 시각화 — Nice-to-have

**동작**: 하위 클래스 선택 시 RightPanel에 "상속된 프로퍼티" 섹션 (접힌 상태, 읽기 전용, 출처 표시)

## 3. LLM 활용 고도화

> **담당**: 온톨로지 전문가 + 테크리더 합의

### 3.1 RightPanel AI 탭 활성화 — Must-have

**현재**: UI만 존재, 기능 미연결 (toast "현재 이 기능은 준비 중입니다")

**구현 (Vercel AI SDK `useChat` 기반)**:
- 선택된 노드의 컨텍스트를 인식한 대화형 AI 보조
- "PM 주기 관련 프로퍼티 추가해줘" → LLM이 현재 클래스 참조하여 제안
- 제안 항목을 클릭하여 바로 적용 (프로퍼티, 관계, 서브클래스 등)
- tiptap 에디터로 리치 입력

### 3.2 대량 문서 파싱 — Must-have

**지원 포맷 (v3)**:
- **CSV/TSV**: 헤더 → 프로퍼티, 행 → 인스턴스 자동 매핑 + 프리뷰
- **자유 텍스트 (대량)**: 청크 분할 → Vercel AI SDK `streamObject`로 병렬 파싱 → 결과 병합
- **클립보드 붙여넣기**: 기존 미구현 버튼 활성화

**UI**: NewNodePopover에 [빠른 입력] / [텍스트 입력] / [CSV] 3탭 분리

### 3.3 자연어 쿼리 → Cypher 변환 — Must-have

**동작**: Toolbar "AI 어시스턴트" 버튼 활성화 → 자연어 질문 → LLM이 온톨로지 스키마 참조 → Cypher 생성 → 프리뷰 → 실행 → 결과를 테이블/서브그래프로 시각화

**구현**: Vercel AI SDK + Neo4j API Route 확장

### 3.4 온톨로지 자동 완성 / 추천 — Nice-to-have

**동작**: 새 클래스 생성 시 AI 탭에 프로퍼티/관계 제안 자동 표시
**주의 (테크리더)**: debounce(500ms+) + 명시적 트리거(버튼/Ctrl+Enter). 자동완성은 로컬 fuzzy match로 처리.

### 3.5 도메인별 온톨로지 템플릿 — Nice-to-have

EmptyState에서 "템플릿으로 시작" 옵션:
- 반도체 제조, IT 인프라, 조직/인사, 의료, 공급망 5종
- 기본 클래스 + 관계 타입 + 프로퍼티 자동 생성

## 4. 사용자 경험 개선

> **담당**: UX설계자 제안 기반

### 4.1 온보딩 — High Impact, Low Effort

#### 4.1.1 가이드 투어
첫 접속 시 3단계 인터랙티브 오버레이:
1. 빈 캔버스 더블클릭 → 노드 생성
2. 노드 클릭 → 상세 패널 확인
3. 노드 간 드래그 → 관계 연결

#### 4.1.2 EmptyState 개선
- **인라인 텍스트 입력 영역** 배치 → 팝오버 없이 바로 입력 가능
- 4가지 CTA: 텍스트 직접 입력 / 파일 업로드 / 템플릿 / URL
- "더블클릭" 의존도 대폭 감소

### 4.2 노드 생성 플로우 개선 — High Impact, Medium Effort

#### 3모드 분리
| 모드 | 용도 | LLM 필요 |
|------|------|----------|
| **빠른 입력** | 단일 노드 (이름/설명/타입/부모) | 불필요 |
| **텍스트 입력** | 자유 텍스트 → LLM 구조화 | 필요 |
| **CSV** | CSV 붙여넣기/업로드 → 대량 생성 | 필요 |

#### 프리뷰 인라인 편집
- 이름 클릭 → 수정 가능
- 프로퍼티 타입 드롭다운 변경 가능
- 항목 추가 가능

#### 에러 핸들링 강화
LLM 실패 시 사용자에게 명시적 선택지: 재시도 / 로컬 파서 / 빠른 입력 전환

### 4.3 커밋/푸시 플로우 단순화 — High Impact, Low Effort

**용어 변경**:
| 현재 | 개선 | 이유 |
|------|------|------|
| 커밋 | **저장** | 도메인 전문가에게 Git 용어 낯설음 |
| Neo4j 푸시 | **반영** (또는 "그래프에 반영") | 기술 용어 은닉 |

**추가 개선**:
- 자동 저장 옵션 (변경 후 일정 시간 경과 시)
- Cypher 프리뷰를 기본 접힌 상태 (기술 세부사항 은닉)
- 반영(푸시) 성공 시 축하 마이크로인터랙션

### 4.4 그래프 탐색 개선 — Medium Impact, Medium Effort

- **고급 필터**: 타입별(클래스/인스턴스) + 색상별 필터링
- **포커스 모드**: 선택 노드 + N-hop 이웃만 표시
- **줌 레벨 표시**: 우하단 퍼센트 표시
- **우클릭 컨텍스트 메뉴**: 편집/삭제/관계추가/포커스 등

### 4.5 관계 관리 개선 — Medium Impact, Low Effort

- 노드 핸들 가시성 강화 (호버 시 강조)
- 우클릭 → "관계 추가" 메뉴
- AI 관계 추천 (AI 탭 연동)

### 4.6 키보드 단축키 & 접근성

**추가 단축키**:
| 단축키 | 동작 |
|--------|------|
| `Ctrl+K` / `Cmd+K` | 커맨드 팔레트 |
| `Ctrl+N` | 새 노드 (빠른 입력) |
| `Ctrl+S` | 저장(커밋) |
| `Ctrl+Enter` | 반영(푸시) |
| `Ctrl+Shift+F` | 캔버스 검색 |
| `Space` (노드 선택 시) | 패널 열기/닫기 |

**접근성**: aria 속성 보강, 포커스 관리, `prefers-reduced-motion` 대응

## 5. 디자인 시스템 & UI 개선

> **담당**: UI/BX디자이너 제안 기반 | 별도 상세: `04-design-system.md`

### 5.1 디자인 시스템 정비

#### 색상
- Semantic Color Token 추가: `--success`, `--warning`, `--info`, `--ai-primary`
- Surface 계층: `surface-0` ~ `surface-3` + `surface-overlay`
- `--accent`를 `--primary`에서 분리 (보조 강조색)
- 노드 팔레트 3색 확장: `concept`(indigo), `process`(teal), `artifact`(violet)

#### 타이포그래피
- **최소 사이즈 11px 상향** (현행 9px, 10px 제거 — 한국어 가독성)
- 5단계 스케일: caption(11) / body-sm(12) / body(13) / heading-sm(14) / heading(18)
- 한국어 폰트 폴백: **Pretendard Variable** 추가

#### 스페이싱
- 4px 그리드 기반 6단계 토큰: xs(4) / sm(8) / md(12) / lg(16) / xl(24) / 2xl(32)
- 패널 패딩 통일: 수평 16px, 섹션 간 12px, 아이템 간 4px

#### 애니메이션
- framer-motion → `motion` v12 마이그레이션 (GPU 가속)
- 통합 프리셋: `panelSlide`, `nodeEnter`, `collapse`, `overlay`
- `prefers-reduced-motion` 대응

### 5.2 UI 개선

#### 노드 디자인
- ClassNode: 원형 유지 + **타입 아이콘 뱃지** (root=Crown, mid=Layers, leaf=Leaf)
- InstanceNode: 원형 → **둥근 사각형** (형태적 구분)
- 빈 클래스: opacity 0.35→0.55 + "비어있음" 라벨
- 노드 라벨: max-w 72→100px, 2줄 허용

#### 엣지 디자인
- is-a: 실선 + 삼각형 화살표
- instance-of: 점선 + 빈 삼각형
- relation: 실선 + 화살표 + **캡슐 배지 라벨**
- 호버: 양쪽 노드 동시 하이라이트

#### 레이아웃
- 패널 리사이저 + 접기/펼치기 (더블클릭으로 토글)
- **커맨드 팔레트** (Cmd+K): 검색, 명령, 노드 이동 통합

#### Elevation 체계
- 4단계: elevation-0(인라인) → elevation-1(카드) → elevation-2(팝오버) → elevation-3(모달)
- AI 전용: `elevation-ai` (ai-glow 그림자)

### 5.3 브랜딩
- 로고: 연결된 3노드 삼각형
- 시그니처 컬러: violet→blue 그라데이션

## 6. 임포트/익스포트

> **담당**: 온톨로지 전문가 제안 + 테크리더 검증

### 6.1 v3 지원 범위

| 포맷 | 임포트 | 익스포트 | 비고 |
|------|--------|---------|------|
| **JSON** (자체 포맷) | ✅ | ✅ | 우선 구현 |
| **CSV** (인스턴스) | ✅ | ✅ | 대량 파싱 기능과 연동 |
| **JSON-LD** | ✅ | ✅ | 표준 포맷, 상호운용성 |
| **Turtle** | — | ✅ | 읽기 쉬운 RDF 포맷 |
| **OWL/XML** | ✅ (기본) | ✅ (기본) | 부분 매핑 (현재 데이터 모델 범위 내) |

### 6.2 테크리더 주의사항
- OWL 완전 매핑은 불가 → 부분 지원 명시, 매핑 불가 요소는 경고 + 건너뛰기
- v3에서는 JSON 내보내기/가져오기 우선, OWL은 기본 수준만
- `rdflib` 또는 `n3` 패키지 활용

## 8. 명시적으로 v3에서 하지 않을 것

| 항목 | 이유 (테크리더 검증) |
|------|---------------------|
| 실시간 협업 (Yjs/Liveblocks) | 복잡도 대비 단일 사용자 MVP에서 가치 없음 |
| OWL/RDF 완전 매핑 | 매핑 범위 미정의, 부분 지원이 오히려 혼란 |
| 추론 규칙 (RDFS/OWL-lite) | 추론 엔진 구현 복잡도 높음 |
| 네임스페이스 관리 | 단일 프로젝트 안정화 우선 |
| 온톨로지 브랜칭/머지 | Git 패턴 완전 구현은 v4+ |
| 다중 사용자/RBAC | Supabase Auth + RLS 전면 재설계 필요 |
| 그래프 시각 diff | 비교 알고리즘 + 렌더링 복잡도 높음, 텍스트 diff 강화로 대체 |
| SPARQL 엔드포인트 | 온톨로지 세계 표준이나 현재 사용자 니즈와 거리 |
| Tailwind v4 + Next.js 16 동시 마이그레이션 | 리스크 분산, v3 안정화 후 |
