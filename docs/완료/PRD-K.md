# Ontology Studio — PRD-K: 핵심 여정 UI 친화화 (Friendly Core Journey)

> **문서 성격**: 목적 · 기능 · UX 설명 중심. 코드·구현·스키마는 제외(구현은 Claude Code). 기존 컴포넌트를 **교체가 아니라 개선**하는 관점 — 컴포넌트명·파일:줄번호는 마이그레이션 기준점으로만 인용.
> **버전**: K-1.0 · 2026-07-06
> **선행/기반**: PRD-I(공통 컨펌 문법·Guided 여정·디자인 토큰) 완료 상태를 전제로 그 위에 얹는다. `온톨로지-스튜디오-UX개선-방향.md`(2026-06-26)의 신뢰 루프(G1)·멘탈 모델(G2) 개선과 상보적 — 본 PRD는 **가시성·크기·스크롤·피드백** 등 물리적 사용성 갭에 집중한다.
> **분석 방법**: 핵심 사용자 여정 2개(① 더블클릭 입력→AI 구조화→컨펌, ② 상세 수정→커밋/발행)를 코드 전수 추적. 문제 33건을 파일:줄번호 근거로 확보.

## 0. 목적

**한 줄 요약**: 킬러 기능(텍스트→AI 구조화→검토·승인)은 강하지만, 그 여정의 UI가 **작고(9~11px), 숨어 있고(hover 전용·2번째 탭), 겹겹이 스크롤되고(3중 중첩), 피드백이 없다(확정 무음·저장 버튼 소멸)**. 비전문가 도메인 전문가가 "지금 내가 어디에 있고, 뭘 눌러야 하고, 방금 한 게 됐는지"를 화면만 보고 알 수 있게 만든다.

**대상 사용자**: 온톨로지·그래프 지식이 없는 도메인 전문가(CLAUDE.md 전제). 마우스 정밀도·hover 발견 능력을 가정하면 안 된다.

**성공의 정의**:
- 첫 사용자가 안내 없이 "더블클릭 → 텍스트 붙여넣기 → 초안 생성 → 확정"을 완주한다.
- 프리뷰에서 원치 않는 항목을 체크 해제로 제외할 수 있다(hover 휴지통 탐색 불필요).
- 수정 작업 중 "저장됐는지"를 패널 안에서 즉시 확인한다(화면 반대편 커밋바를 보지 않고).
- 본문 폰트 최소 12px, 클릭 타깃 최소 24×24px(WCAG 2.2 Target Size 최소 기준).

## 3. 개선 설계

### 3-A. 전역 스케일 업 — "9px 금지" 규칙 (M1)

디자인 토큰 차원에서 최소 크기를 강제한다. 신규 화면이 아니라 **기존 화면의 밀도 재조정**이다.

- **타이포 스케일 재정의**: 본문 최소 `text-xs`(12px) → 표준 `text-sm`(13~14px). `text-[9px]`·`text-[10px]`은 **전면 금지**, 배지·캡션 하한 11px. 섹션 제목은 본문보다 한 단계 크게(12px 본문 ↔ 14px 제목) — 굵기가 아니라 크기로 위계.
- **타깃 스케일**: 클릭 가능한 모든 요소 최소 24×24px(WCAG 2.2), 주요 폼 컨트롤(input/select/버튼) 높이 32px(h-8) 표준. 20px(w-5 h-5) 버튼 폐지.
- **hover 전용 노출 폐지**: `opacity-0 group-hover:opacity-100` 패턴을 "상시 저채도(muted) → hover 시 강조"로 전환. 터치·발견성 동시 해결. (프리뷰 삭제/전환, RightPanel 연필/삭제/오버라이드 전부)
- **적용 범위**: NewNodePopover 프리뷰, RightPanel 전체, CommitBar, ConfirmCard, LifecycleIndicator. 토큰/클래스 치환 중심이라 로직 변경 없음.

### 3-B. 입력 진입 — "더블클릭하면 바로 붙여넣는다" (M2)

- **더블클릭 = 자연어 우선**: 더블클릭 진입 시 기본 탭을 `text`(자연어)로. Quick(단일 노드 폼)은 두 번째 탭으로 이동 — 킬러 기능을 첫 화면에. (짧은 이름 한 단어만 치면 시스템이 "빠른 추가로 충분해요" 힌트로 Quick 유도 — 강제 없음)
- **노드가 있는 캔버스에도 배경 더블클릭 바인딩**: EmptyState 전용이던 진입을 Cytoscape 배경 더블클릭에도 연결. 어느 상태든 같은 제스처 = 같은 결과.
- **죽은 버튼 처리**: "파일/붙여넣기" 버튼은 구현(파일 선택→텍스트 삽입, 클립보드 읽기)하거나 제거. 동작하지 않는 UI 잔존 금지.
- **정직한 로딩**: 진행 단계를 실제 API 단계(엔티티 추출→관계 추론)와 동기화. 인위적 % 대신 "1/2단계: 엔티티 추출 중… (12s)" 식 단계+경과시간. 100자 미만 입력도 동일한 로딩 상태 표시(임계값에 따른 피드백 유무 차별 제거).
- **로딩 중 취소·입력 보존**: 취소하면 입력 텍스트가 그대로 남은 입력 화면으로 복귀(현행 유지·명문화).

### 3-C. 프리뷰·컨펌 — "체크하고, 확정하고, 확인받는다" (M3, 본 PRD의 척추)

**원칙: 검토는 팝오버가 아니라 검토 표면에서.** 소량(예: 클래스 3개 이하·관계 5개 이하)은 현행 팝오버 프리뷰 유지, 그 이상은 **넓은 검토 표면**(바텀 Sheet 또는 중앙 확대 표면 — 기존 Sheet 관습 계승)으로 자동 승격. GuidedJourney를 새로 만드는 게 아니라 **프리뷰 단계 하나를 넓은 표면에 옮겨 담는 것.**

1. **항목별 체크박스 포함/제외**: 파싱된 클래스·속성·인스턴스·관계 각각 체크박스(기본 전체 선택). "확정"은 체크된 것만 반영. hover 휴지통 탐색 불필요. 반영 모델을 "핵심도 부가도 전부 opt-out 가능한 선택 반영"으로 통일.
2. **요약 헤더 상시 고정**: "클래스 5 · 속성 12 · 관계 8 선택됨 — 2건 제외" 를 검토 표면 상단에 sticky. 스크롤해도 전체 그림 유지.
3. **이탈 가드**: 프리뷰 상태에서 배경 클릭/Esc 시 "생성된 초안 12건을 버릴까요? [계속 검토] [버리기]" 확인. 실수 1클릭으로 LLM 결과 전량 소실 방지.
4. **확정 피드백**: 확정 시 ① 성공 토스트("클래스 5, 관계 8 추가됨 — 되돌리기") + ② 새 노드 캔버스 하이라이트(기존 모션 토큰 사용) + ③ CommitBar 카운트 증가 애니메이션. "조용히 나타나는" 현행 제거.
5. **부가 검수의 단계화**: Critic·중복검사·거버넌스·보강을 우측 260px 적층에서 꺼내 **검토 표면의 순차 스텝**(구조 확인 → 중복 확인 → 규칙 제안 → 확정)으로. 각 스텝은 기존 ConfirmCard 그대로, 스테퍼는 기존 JourneyStepper 문법 재사용. "건너뛰고 바로 확정" 항상 가능.
6. **단계 표시**: 입력→분석→검토→확정 4단계를 프리뷰 상단에 미니 스테퍼로. 팝오버/여정 이원화된 단계 안내를 통일.

### 3-D. 상세 수정 — "저장됐는지 그 자리에서 안다" (M4)

1. **탭 고정 6개**: 선택 없을 때도 탭 구성 유지(비활성 탭은 dim + "노드를 선택하세요" 빈 상태). 3↔6 변동 제거.
2. **스크롤 단층화**: 패널 스크롤은 ScrollArea 1개로. 내부 테이블·섹션의 자체 스크롤 제거(콘텐츠가 길면 섹션 접기·"더 보기"로 해소). "지금 어디를 스크롤하고 있는가" 혼란 제거.
3. **필드 저장 피드백**: blur 저장 성공 시 필드 옆 체크 아이콘 1.5초 표시 + 패널 헤더에 "초안에 저장됨 ✓" 상시 상태(자동저장 인디케이터를 패널로 확장). 실패 시 인라인 에러.
4. **인스턴스 테이블 실값 표시**: `—` 하드코딩을 실제 값 미리보기로.
5. **관계 추가 팝오버 위치**: 화면 정중앙이 아니라 패널 인근(트리거 버튼 기준)으로.
6. **섹션 위계**: 제목 14px/본문 12px 크기 대비 + 저채도 아이콘 상시 노출(3-A 규칙 적용).

### 3-E. 확정 여정 가시화 — "초안 → 검토 → 발행"이 항상 보인다 (M5)

1. **상태 문장 상시 표기**: CommitBar 좌측에 "지금: **초안 편집 중** · 변경 7건 · 발행 안 됨" 같은 평문 한 줄(툴팁 승격). LifecycleIndicator는 유지하되 크기 상향(3-A).
2. **자동저장 On에서도 저장 상태 상시 표시**: 버튼 소멸 대신 "✓ 모두 저장됨 (방금)" 상태 텍스트로 대체. 클릭하면 즉시 저장 실행(수동 트리거 보존).
3. **"반영" → "발행" 언어 통일** + 발행 버튼에 사전 요약("클래스 5 · 관계 8을 운영 그래프에 발행") 확인 스텝 — 기존 NeoConfirmSheet 흐름에 요약 헤더만 보강.
4. **툴바 그룹핑**: 20개+ 균일 나열을 4그룹(보기 · 편집 · 품질 · AI)으로 시각 구분(구분선+간격). 저빈도 관리 액션(건강도·중복검사·병합)은 "품질" 드롭다운으로 접기. 버튼 수 자체를 줄여 남는 것을 키운다.
5. **undo/redo 단일화**: 툴바로 일원화(undo+redo 나란히), CommitBar "되돌리기"는 "변경 전체 취소(확인 필요)"로 역할 분리·명명.

## 4. 디자인 원칙 (PRD-I 계승 + 추가)

1. **토큰 단일화·primitive 재사용·ConfirmCard 문법** — PRD-I §1 그대로. 새 라이브러리·새 팔레트 금지.
2. **(신규) 최소 크기 규칙**: 본문 12px↑ · 캡션 11px↑ · 타깃 24px↑ · 폼 컨트롤 32px 표준. 이 규칙을 어기는 기존 클래스는 발견 즉시 치환 대상.
3. **(신규) 상시 가시 원칙**: 상태(저장·단계·모드)는 툴팁·hover가 아니라 화면에 상시 텍스트로. hover는 강조용이지 존재 은닉용이 아니다.
4. **(신규) 피드백 3종 세트**: 모든 확정 액션은 ① 즉시 확인(토스트/체크) ② 위치 표시(하이라이트) ③ 되돌리기 경로를 함께 제공.
5. **교체 0, 확장·재조정 위주**: 스토어 액션·API·검수 로직은 불변. 표면(크기·배치·피드백)만 손댄다.
