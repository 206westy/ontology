# 그래프 UX 리브랜딩 + AI 컨펌형 작성 사용성 개선 (완료)

> 상태: **완료·라이브 검증됨** (마일스톤 1~5 전부)
> 원본 기획: `.claude/prds/ontology-graph-ux.prd.md` (로컬 전용, 미추적)
> 성격: 신규 기능이 아니라 **기존 기능의 개선** — 자연어→클래스/인스턴스/관계/제약 추출→미리보기→컨펌 흐름은 이미 구현돼 있었음(`NewNodePopover`).

## 문제

비전문가가 그래프만 보고 노드·선의 의미(클래스/인스턴스/계층/관계)를 구분하지 못하고, 세부(서브클래스/속성/제약/인스턴스)는 패널에 숨어 채우기 어려워 온톨로지가 빈 껍데기로 남는다. CSV 적재는 4컬럼 고정 파싱뿐이라 실데이터를 의미 있게 담지 못했다.

## 분류 원칙 (클래스 vs 인스턴스 vs 서브클래스)

축은 "관계 유무"가 아니라 **스키마(일반 규칙/틀) vs 데이터(구체 사실)**. 인스턴스도 관계를 가진다(오히려 최다). 결정적 판별식 = `instance_values`: 값 슬롯을 채우는 레코드 → 인스턴스, 아래 것의 속성/관계/제약을 정의·좁히는 틀 → 클래스/서브클래스. **기본값=인스턴스**(서브클래스는 구조적 차이가 있을 때만). 얕은 계층(Foundry식 flat object type). 일반 규칙("모든 펌프는 씰을 가진다")→Constraint/Axiom, 특정 사실("펌프447-씰9982")→인스턴스 edge.

## M5 CSV 설계

- **데이터 자체 설명**: 메인 객체타입 1클래스 + 컬럼별 dataType 추론(integer/float/boolean/date/enum/string) + 각 행 → 인스턴스(전 컬럼을 값으로 보존).
- **인사이트**: 참조(FK형) 컬럼 → 별도 클래스 + distinct 값 인스턴스(중복제거) + 행→참조엔티티 structural 관계, 범주 컬럼 → enum 속성. 측정/숫자/자유텍스트 → 속성값(노드화 금지). 같은 행의 참조 링크는 co-occurrence가 아니라 FK=근거로 취급.
- **길이 제한**: 행 수 무관, 내부 전체 텍스트 15,000자.
- 라이브: 설비 6행 CSV → 설비/공급사/부서 클래스, EQ-001~006 인스턴스(상태=enum, 정격출력=float, 도입일=date), supplied_by/belongs_to structural 관계, 측정값 노드화 0.
