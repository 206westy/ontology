# 지식 입력 · AI 컨펌형 작성

[← README](../../README.md) · 관련: [그래프 시각 언어](02-graph-visual-language.md) · [AI Critic·거버넌스](05-ai-critic-governance.md)

도메인 지식을 자유 형식으로 쏟아내면 AI가 온톨로지 구조로 정리하고, 사용자는 **미리보기에서 검토·수정 후 확정**합니다. AI는 초안을 만들 뿐, 반영 권한은 사람에게 있습니다(HITL).

![지식 입력](../assets/knowledge-input.png)

## 진입 방법

- 캔버스 **빈 공간 더블클릭** → 클릭한 위치에 입력 팝오버
- 툴바 **가져오기** 버튼
- 노드 **우클릭 → 새 클래스**
- 첫 화면(빈 캔버스)의 안내 카드

팝오버에는 세 가지 입력 탭이 있습니다: **빠른 입력**(노드 1개 직접) · **텍스트 입력**(자유 형식) · **CSV**([별도 문서](04-csv-ingestion.md)).

## 2단계 추출 (왜 나눴나)

자유 텍스트는 한 번의 호출이 아니라 **두 단계**로 처리합니다.

1. **Stage 1 — 엔티티(점)**: 개념·부품·파라미터·측정값을 추출하고, 각각을 클래스/인스턴스로 분류하며 속성 값을 채웁니다.
2. **Stage 2 — 관계(선)**: 추출된 엔티티 사이에서 텍스트에 근거가 있는 관계만 뽑습니다.

단계를 나눈 이유는 단일 호출이 만들던 **"문서 제목이 허브가 되는 별모양"** 편향을 없애고, 근거 없는 관계 대신 정직한 "섬"(고립 노드)을 허용하기 위해서입니다.

### 분류 원칙 — 스키마 vs 데이터

클래스/인스턴스/서브클래스를 가르는 축은 "관계가 있냐"가 아니라 **일반 규칙(틀)이냐, 구체 사실(레코드)이냐**입니다. 결정적 판별식은 *값 슬롯을 채우는가*:

- **인스턴스** — 값 슬롯을 채우는 구체 레코드(시리얼/고유명사/특정 시점의 개체). 기본값.
- **클래스** — 아래 것의 속성·관계·제약을 정의·좁히는 틀.
- **서브클래스** — 부모와 **구조적으로 다를 때만**(값만 다르면 속성, 구체 개체면 인스턴스).

계층은 얕게(Foundry식 flat Object Type) 둡니다.

## 미리보기 · 컨펌

추출 결과는 확정 전 미리보기로 표시됩니다.

- **계층 트리** — 클래스/인스턴스를 들여쓰기로, 기존 노드는 흐리게(재사용 표시)
- **클래스 ↔ 인스턴스 전환** — AI 분류를 사용자가 즉석 교정
- **속성 값·관계** — 인스턴스의 추출된 값과 관계를 함께 노출(비가시 저장 방지)
- **검수·중복·보강** 패널 — [AI Critic·거버넌스](05-ai-critic-governance.md) 참고

대량 입력(100자 이상)은 단계별 진행률이 표시되고 취소할 수 있으며, 1회 호출 상한(텍스트 8,000자)을 넘으면 나눠 입력하도록 막습니다.

## 출처(Provenance)

확정된 노드·관계에는 추출 근거(evidence)와 출처 유형·confidence가 함께 저장되어, 나중에 "이건 어디서 나온 사실인가"를 추적할 수 있습니다.
