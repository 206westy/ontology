# AI Critic · 거버넌스 · 보강

[← README](../../README.md) · 관련: [지식 입력](01-knowledge-input.md) · [온톨로지 Git](06-staging-commit-push.md)

AI의 역할은 "새 그래프를 그리는 것"이 아니라 **지속되는 단일 모델을 깨끗하게 지키는 것(Critic)**입니다. 새 입력이 확정되기 전에 자동 검수가 돌고, 모든 판단은 제안일 뿐 확정을 막지 않습니다.

![검수가 포함된 미리보기](../assets/ai-preview.png)

## 검수 (Critic)

미리보기 우측에 검수 리포트가 함께 뜹니다. 결정론적 규칙을 클라이언트에서 직접 돌려(네트워크 0, 실패 모드 0) 다음을 플래그합니다.

- **클래스/인스턴스 혼동** — 스키마와 데이터가 뒤섞인 경우
- **별모양 허브 / 고아 노드** — 한 노드에 관계가 비정상 집중, 또는 완전 고립
- **0-정보 관계** — 같은 문서에 나왔다는 이유뿐인 근거 없는 관계
- **중복·동의어** — 이미 모델에 있는 개념의 재등장
- **미정의 개념** — 정의 없이 참조만 되는 이름

심각도(높음/중간/낮음)로 정렬되며, 각 이슈는 **무시** 가능합니다(읽기전용 자문).

## 중복 대조 (Dedup)

추가하려는 신규 노드를 기존 노드와 **의미 + 오타** 양면으로 대조합니다. 임베딩 후보 검색 → LLM 판정으로 `재사용 / 관계 / 중복 가능 / 신규` 중 하나를 제안합니다. 재사용으로 판정되면 새로 만들지 않고 기존 노드에 연결합니다.

## 거버넌스 제안 (HITL)

텍스트 근거를 바탕으로 **제약(Constraint)·공리(Axiom)**를 제안받습니다. 일반 규칙("모든 펌프는 씰을 가진다")은 인스턴스 간선이 아니라 거버넌스로 기록되어야 한다는 원칙을 따릅니다. 제안은 **승인 전 미반영**이며, 승인 시에만 제약/공리 테이블에 출처와 함께 기록됩니다.

## 보강 (Enrichment)

추출된 서브그래프에서 정의 누락·고립·빈약한 노드 등의 **갭**을 탐지하고, 내부 → 세션 문서 → (옵트인) 웹 순으로 근거를 소싱해 채택을 제안합니다. 웹 검색은 기본 꺼짐이며 "검증 필요"로 표시됩니다.

## 모델 헬스

모델 전체의 충실도·일관성을 **상시 점수**로 노출해(툴바 건강도), 입력 전후로 "성능"이 어떻게 변하는지 눈에 보이게 합니다.
