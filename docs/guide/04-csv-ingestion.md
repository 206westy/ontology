# CSV 분석

조직의 표 데이터는 그 조직의 **필수 데이터**입니다. CSV 탭은 단순 적재가 아니라, 표를 **데이터 자체를 설명하는 온톨로지 + 데이터에서 얻는 인사이트**로 변환합니다.

## 어떻게 읽나

CSV는 **첫 줄 = 헤더(스키마), 각 행 = 레코드**로 읽습니다. 텍스트 입력과 같은 2단계 파스/미리보기/컨펌 파이프라인을 쓰되, CSV 전용 프롬프트로 동작합니다.

### (a) 데이터 자체를 설명하는 온톨로지
- **메인 객체타입 1개** — 각 행이 무엇인지(설비, 환자, 거래…)를 클래스로
- **컬럼 → 타입 추론** — `integer / float / boolean / date / enum / string`
- **각 행 → 인스턴스** — 모든 컬럼 값을 그 인스턴스의 속성 값으로 보존

### (b) 데이터에서 얻는 인사이트
- **참조(FK형) 컬럼 → 별도 엔티티** — 공급사·부서·담당자·위치처럼 *독립 정체성을 가진* 컬럼은 별도 클래스로 만들고, 서로 다른 값들을 인스턴스로(중복 제거)
- **행 ↔ 참조 엔티티 관계** — 같은 행의 참조는 표의 외래키 → `structural` 관계로 연결(예: 설비 →`supplied_by`→ 공급사)
- **범주 컬럼 → enum 속성** — 상태/등급/구분처럼 값이 몇 개로 반복되면 노드가 아니라 enum 값
- 측정·숫자·자유 텍스트 → **속성 값**(절대 노드로 만들지 않음)

## 예시

```csv
설비ID,설비명,공급사,부서,상태,정격출력(kW),도입일
EQ-001,식각기 1호,삼성전자,식각팀,가동,5.5,2021-03-01
EQ-002,증착기 2호,램리서치,증착팀,정지,12.0,2020-07-15
```

→ **클래스**: `설비`(메인), `공급사`·`부서`(참조)
→ **인스턴스**: `EQ-001…`(상태=enum, 정격출력=float, 도입일=date), `삼성전자`·`식각팀`…
→ **관계**: `EQ-001 →supplied_by→ 삼성전자`, `EQ-001 →belongs_to→ 식각팀`

## 길이 제한

행 수가 아니라 **내부 전체 텍스트 기준 15,000자**로 제한합니다(입력창에 실시간 글자수 표시). 한 번의 호출에 표 전체가 들어가도록 한 상한입니다.
