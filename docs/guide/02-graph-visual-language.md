# 그래프 시각 언어

[← README](../../README.md) · 관련: [지식 입력](01-knowledge-input.md) · [속성 패널](03-property-panel.md)

설명 없이도 노드·선의 의미가 한눈에 읽히도록 그래프 표현을 재설계했습니다. 캔버스 좌하단 **범례**에 항상 노출됩니다.

![범례와 그래프](../assets/hero.png)

## 노드 — 클래스 vs 인스턴스

| 표현 | 의미 |
|------|------|
| **큰 원** + 이름 | 클래스(유형/카테고리) |
| **작은 점** | 인스턴스(구체 사례) — 호버 시 이름, 클릭 시 패널 |
| **점선 원** | 빈 클래스(아직 인스턴스/세부 없음) |

색은 도메인(루트/중간/하위/사람/장소/이벤트/개념/산출물)을, 크기는 연결 차수를 나타냅니다.

## 선 — 계층 vs 소속 vs 관계

| 표현 | 의미 |
|------|------|
| **파선 →** | 상-하위(계층, is-a) |
| **점선** | 소속(인스턴스 → 클래스) |
| **실선** (이름표) | 관계(액션 지향: 구조/인과/진단/절차) |

관계는 액션 지향 분류로 색을 달리해, 단순 서술이 아니라 "무엇을 할 수 있는 관계"인지 구분합니다.

## 인스턴스 점 표현 + 확장성

인스턴스를 풀 노드가 아닌 **점**으로 그려 한 클래스에 사례가 많아도 화면이 어지럽지 않습니다.

- **호버** → 이름 표시 · **클릭** → 우측 속성 패널
- 한 클래스의 인스턴스가 **20개를 초과하면 기본 접힘** — 클래스 라벨에 `이름 (N)`으로 개수 표기
- **클래스 더블클릭** → 인스턴스 펼침/접기 토글
- 접힌 인스턴스는 렌더링·레이아웃 물리 계산에서 동시에 제외되어, **200개 이상에서도 부드럽게** 동작

## 줌 레벨에 따른 표시 (LOD)

| 줌 | 표시 |
|----|------|
| 높음 | 이름 + 세부 |
| 중간 | 이름 |
| 낮음 | 색상 점만 |

전체 구조 파악과 세부 탐색을 같은 캔버스에서 오갈 수 있습니다.
