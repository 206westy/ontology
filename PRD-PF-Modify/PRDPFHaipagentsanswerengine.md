# PRD-PF-H — AIP 기능: 답변엔진 · 진단형 RAG · 제안형 에이전트 (AIP Logic/Assist/Agents 대응)

> **제품:** 온톨로지 스튜디오 — 코드 없이 온톨로지 구축·운영 SaaS · AI=Critic(제안+사람 컨펌)
> **분류:** 소비 표면(P3 Operator) — 온톨로지를 "믿을 수 있는 답변·진단·제안" 엔진으로

> **작성일:** 2026-07-12
> **원칙:** 비판적 옹호 · 기존 소비 자산 재배치 우선 · shadcn/ui · 한국어 UI · HITL · 읽기전용 가드레일 · 시각언어(원/점/파선/실선) 보존 · 과대약속 금지

## 0. 한 줄 요지

우리는 온톨로지를 **만드는 절반**(파싱→Critic 9룰→커밋→발행)은 성숙했다. 이 PRD는 **소비하는 절반** — 팔란티어 AIP(Logic/Assist/Agents)가 파는 "온톨로지 위에서 답하고·진단하고·제안하는 표면" — 을 세우되, AIP의 핵심 정직 원칙(**에이전트는 자율실행이 아니라 proposal+사람 검토**)을 그대로 취해, 이미 가진 자산(구획=커뮤니티, provenance=인용, Critic=신뢰, `api/rag/entrypoint` 스캐폴드)을 재배치해 **감사 가능한 모델 위에서 근거경로로 답하는** GraphRAG 답변엔진·진단형 RAG·제안형 에이전트를 만든다.

---

## 1. 목적 (Purpose)

**왜 지금 AIP 대응인가.** 팔란티어 AIP는 온톨로지 위에 세 층의 소비 표면을 얹어 "지식 그래프"를 "운영 시스템"으로 전환하고, 바로 그 지점에서 매출을 낸다:

- **AIP Logic** — 자연어 목표를 온톨로지 위 함수·데이터를 엮은 **추론 체인**으로 실행.
- **AIP Assist** — 플랫폼 전반을 자연어로 조작·안내하는 보조.
- **AIP Agents** — 에이전트가 상황을 감지·분석해 조치를 **제안(proposal)** 하고, 사람이 검토·확정. **자율실행이 아니다.**

**결정적 재해석(우리 관점).** AIP가 파는 것은 "그래프 위에서 답하고 행동을 제안하는 표면"이지, "AI가 알아서 실행하는 자동화"가 아니다. 팔란티어조차 액션은 사람 확정(HITL)을 거친다. **이것은 위협이 아니라 우리 북극성(AI=Critic, 제안+컨펌)과 정확히 같은 철학이며, 우리에게 유리한 지형이다.** 우리는 (a) 사람이 큐레이션하고 Critic이 검수한 **깨끗한 모델**, (b) 이미 나뉜 **구획**(=GraphRAG 커뮤니티), (c) 커밋마다 붙는 **provenance**(=인용), (d) 완성된 **Text2Cypher 읽기전용 가드**(`findWriteClauseViolation`), (e) **pgvector 임베딩 검색**, (f) **RAG 진입 스캐폴드**(`api/rag/entrypoint`: 질문 임베딩→Neo4j 벡터인덱스 top-k 진입노드)를 이미 보유한다. 즉 AIP가 데이터 파이프라인부터 무겁게 깔아야 얻는 것을, 우리는 **재배치로 얻는다.**

**우리 데이터 기반 RAG의 차별점(핵심 세일즈).** 경쟁 GraphRAG는 LLM이 자동 추출한 **노이즈 그래프**에 답을 얹는다("쓰레기 넣으면 쓰레기 요약"). 우리는 **사람이 검수(Critic)한 감사 가능한 단일 모델** 위에서 **근거경로로 인용**해 답한다. 규제·안전 산업(설비보전·품질·공정)일수록 이 "왜 이 답이 나왔는가"의 추적성이 구매 요인이다.

**비판적 옹호(경계).** 우리는 AIP Logic의 **TypeScript 함수 저작 IDE를 재구현하지 않는다.** GraphRAG 인덱서(Leiden 군집화·엔티티 추출·전량 요약)를 처음부터 짓지도 않는다. 우리 할 일은 **이미 가진 구조를 답변/진단/제안 인터페이스로 노출**하는 것. 그리고 가장 중요한 정직 경계: **완전자동 금지 — 에이전트는 제안까지, 실행 확정은 사람. 근거 없으면 "모델에 근거 없음". 소비는 읽기전용(모델 변경은 별도 생성흐름).**

## 3. 기술 스택 (기존 소비 자산 재배치 우선 — 신규 인프라 최소)

AIP의 비싼 절반(파이프라인·인덱싱·함수 IDE)을 우리는 다른 이름으로 이미 가지고 있거나, 별도 흐름(PRD-B/G)에 위임한다.

| AIP 개념 | 우리의 대응 자산 | 상태 | 이 PRD의 보강 |
|---|---|---|---|
| 온톨로지(객체·속성·링크) | 파싱 파이프라인 + Neo4j 발행 그래프 | ✅ 보유 | — |
| 커뮤니티/맥락 군집 | **구획(`partitions`)** — 사람+AI 명시 경계 | ✅ 보유 | 커뮤니티 재사용(군집화 재구현 불필요) |
| 커뮤니티 요약 | 구획별 요약 텍스트 | 🔴 미보유 | **M1** — 커밋 시 dirty 구획만 생성 |
| 로컬 검색(엔티티 이웃) | `api/rag/entrypoint`(질문 임베딩→Neo4j 벡터인덱스 top-k 진입노드) | 🟡 진입만, 합성 미완 | **M2** — 스코프+경로 확장+답변 합성 |
| 글로벌 검색(map-reduce) | — | 🔴 미보유 | **M3** — 구획 요약 map-reduce |
| Cypher 조회·읽기전용 가드 | **Text2Cypher 완성**(`api/llm/text2cypher`, `findWriteClauseViolation`) | ✅ 보유 | 진단 경로 쿼리·가드레일 재사용 |
| 후보 소환 임베딩 | **pgvector 임베딩 검색** | ✅ 보유 | 앵커·구획 요약 검색에 배치 |
| AIP Logic(추론 체인) | Critic 9룰 규칙엔진 + 결정함수(PRD-B) | 🟡 부분 | **M4** — 자연어→온톨로지 위 추론체인(근거 노출), 함수 저작은 PRD-B 위임 |
| AIP Assist(자연어 조작) | Text2Cypher·기존 스튜디오 액션 | 🟡 | **M4** — 스튜디오 조작 보조(읽기·탐색 우선) |
| AIP Agents(제안형) | HITL·Critic(제안+컨펌) 사고 | 🔴 | **M5** — 감지→분석→**제안**, 액션보드는 PRD-G 위임 |
| 인용·출처(provenance) | `source_type`/`confidence`/`origin_commit` | ✅ 보유 | **M2~M5** — 근거경로로 노출 |
| 가드레일(읽기전용) | `findWriteClauseViolation`·RLS·HITL·Critic 9룰 | ✅ 보유 | 소비·에이전트 전 표면 읽기전용 불변식 |

**스택 정합:** Next.js15/React19/TS5 · Supabase(Drizzle·pgvector, SoT) · Neo4j6(발행·벡터인덱스·경로) · Vercel AI SDK v6+@ai-sdk/openai v3(tool-calling·에이전트 오케스트레이션) · Cytoscape(인용 하이라이트) · zod4(도구 인자 검증) · shadcn/ui(질문·제안 카드). 원칙: **AIP가 확률적으로 복원하는 구조를 우리는 사람이 승인한 형태로 결정론적으로 조회**한다.

## 4. 방향 (마일스톤 로드맵)

> 순서 근거: **요약 재료(M1) → 국소·진단(M2, 스캐폴드 완성) → 전역(M3) → 추론체인·조작 보조(M4) → 제안형 에이전트(M5)**. 전 마일스톤 읽기전용이라 회귀 위험 낮고 각 단계가 단독 가치를 준다.

### M1 — 구획 요약(Community Summaries) · 소비 표면의 척추
- 각 **구획**에 LLM이 *"핵심 클래스·관계·대표 인스턴스·경계"* 요약 텍스트 + 임베딩(pgvector) 생성.
- **트리거 = 커밋(재사용).** 새 스냅샷 시 **변경된 구획만** 재요약(전량 재계산 금지). is-a 계층 따라 하위→상위 롤업 요약 선택 생성.
- 구현: `summaries` 테이블(`partition_id`·`commit_id`·`summary`·`embedding`·`stale`) + `POST /api/summary/rebuild`(구획 단위). 커밋 훅에서 dirty 구획만 마킹.
- **Critic 결합(해자):** 요약에 그 구획 Critic 헬스(별모양·설계위반 지수) 스탬프 → 답변 신뢰도에 반영.

### M2 — 국소·진단형 RAG · `api/rag/entrypoint` 스캐폴드를 완성으로 (핵심)
- 현재: 질문 임베딩→Neo4j 벡터인덱스 top-k **진입노드만**. 여기서 멈춰 있음.
- 완성: 진입노드 → **이웃 확장 + 경로 추적**(Text2Cypher 읽기전용 쿼리 재사용, `findWriteClauseViolation` 가드) → LLM 답변 합성 → 근거경로 부착.
- **구획 스코프 기본**: 무관 구획 triple 오염 방지, "전체 질의"는 opt-in.
- **진단형 RAG(운영 도메인=설비보전 예시):** *"3호기 진공펌프 반복 고장의 근본원인 후보는?"* → 인스턴스 앵커 → 고장→조치 경로 추적 → 근거와 함께 **원인후보·조치추천(정비/공정)** 제시. **행동은 제안까지, 확정은 HITL.**
- UI: 기존 `Text2CypherTab` 옆 **"질문하기(Ask)"** 탭 — 평문 입력, 답변 + 접힌 "근거" 칩(shadcn/ui, `<ConfirmCard>` 톤).

### M3 — 전역 검색(Global Search) · 사센스메이킹
- 전역 질의("전체에서 ○○ 패턴은?", "구획 간 공통점은?")면 구획 요약을 **map-reduce**로 종합.
  - *map*: 관련 구획 요약별 부분 답변 + 근거 구획 id.
  - *reduce*: 부분 답변 종합.
- **질의 라우팅**: 결정론 힌트(전체/구획명/"패턴·경향" 키워드) + 소형 LLM 분류(전역 vs 국소). 애매하면 DRIFT(둘 다 실행 후 병합).
- **비용 방어**: 라우팅·인용 매핑은 소형 모델, 종합만 상위 모델. 관련 구획만 map(요약 임베딩 사전 선별).

### M4 — AIP Logic/Assist 대응 · 추론 체인 & 자연어 조작 보조
- **Logic 대응(추론 체인):** 자연어 목표 → 온톨로지 위 추론 체인을 **투명하게** 실행: "어느 구획·노드·경로·결정함수(PRD-B)·커밋 근거로" 각 단계를 밟았는지 노출. 함수 저작 자체는 PRD-B, 실행 확정은 HITL로 위임(우리는 근거 있는 추론의 조립·설명에 집중).
- **Assist 대응(조작 보조):** 스튜디오 전반 자연어 안내·탐색 보조. **읽기·탐색·설명 우선**, 모델 변경 제안은 반드시 기존 파싱/Critic/HITL 흐름으로 우회.
- 온도 낮춤·tool-calling(AI SDK v6)·근거 없으면 "모델에 근거 없음".

### M5 — 제안형 에이전트(AIP Agents 대응) · 자율실행 금지
- 에이전트 루프: **이상 감지 → 원인 분석(M2 진단 RAG 재사용) → 조치 제안 생성**.
- 산출물은 실행이 아니라 **proposal**: 근거경로·영향 범위·신뢰도를 담아 **액션보드(PRD-G)** 에 올림. **엔지니어가 검토·확정.**
- **팔란티어 AIP와 동일 원칙(proposal+review).** 읽기전용 가드레일·환각억제·근거 필수. 구획 스코프 기본.
- 에이전트는 그래프를 1비트도 변경하지 않는다(변경은 확정 후 PRD-G/생성흐름).

## 5. 방법론 (검색 아키텍처 · 에이전트 가드레일 · 리스크)

### 5.1 검색 아키텍처 (AIP 능력 → 우리 매핑)

| AIP/GraphRAG 능력 | 우리 매핑 | 재사용 자산 | 상태 |
|---|---|---|---|
| 커뮤니티 요약 | 구획 요약(M1) — 군집화 불필요 | partitions·커밋훅·pgvector | 🔴→M1 |
| 로컬 검색 | 진입노드→이웃·경로(M2) | `rag/entrypoint`·Text2Cypher·pgvector | 🟡→M2 |
| 글로벌 검색 | 요약 map-reduce(M3) | 구획 요약 임베딩 | 🔴→M3 |
| DRIFT | 국소/전역 병합 라우팅 | 라우터(소형 LLM) | 🔴→M3 |
| 추론 체인(Logic) | 근거 노출 추론 조립(M4) | Critic 9룰·결정함수(PRD-B) | 🟡→M4 |
| 조작 보조(Assist) | 자연어 탐색·설명(M4) | Text2Cypher·스튜디오 액션 | 🟡→M4 |
| 제안형 에이전트(Agents) | 감지→분석→제안(M5) | 액션보드(PRD-G)·HITL | 🔴→M5 |
| 근거·인용 | 근거경로 부착(M2~M5) | provenance·origin_commit | 🟡→전 마일스톤 |

### 5.2 에이전트 가드레일 (정직 경계의 기술적 구현)
1. **읽기전용 불변식:** 소비·에이전트 전 표면이 Cypher 쓰기 절을 못 낸다 — `findWriteClauseViolation` 재사용 + RLS. **계약 테스트로 0건 보장.**
2. **자율실행 금지:** 에이전트 산출은 언제나 proposal 객체(근거·영향·신뢰도). 실행 확정은 사람. 액션보드(PRD-G)만 상태 변경 경로.
3. **환각 억제:** 답변 문장은 인용 없으면 "모델에 근거 없음"으로 강등(침묵 > 지어내기). 온도 낮춤, 전역 답은 구획 요약 컨텍스트로만.
4. **소비↔생성 경계:** 소비가 시사하는 모델 개선은 "이 답 근거로 수정 제안 만들기" → 기존 파싱/Critic/HITL로 위임.
5. **신선도:** `stale` 플래그·dirty 구획만 재요약, 답변에 "요약 기준 커밋" 표기(as-of).

### 5.3 리스크 & 완화
- **R1. 답변엔진 비용(map-reduce LLM 다단 호출).** → 라우팅·인용 매핑 소형 모델, 종합만 상위 모델, 관련 구획만 map, 비용 예산 노출.
- **R2. 환각(최대 신뢰 리스크).** → 근거경로 필수화·"모델에 근거 없음" 강등·읽기전용·저온도.
- **R3. 에이전트 제안 품질.** → 구획 스코프 기본, Critic 헬스 낮은 구획 제안은 신뢰도 하향·경고 배지, 채택률 도그푸딩 계측.
- **R4. 요약 신선도(stale).** → dirty 구획만 재요약·as-of 커밋 표기.
- **R5. 스코프 크리프(GraphRAG 인덱서·AIP Logic IDE 재구현 유혹).** → 군집화·전량 요약·함수 저작 IDE는 **명시적 Out**. 재배치 100%.
- **R6. 소비↔생성 경계 흐림(SoT 무결성).** → 읽기전용 RLS 불변식, 변경은 생성흐름/PRD-G로만.

## 6. 수용 기준 (Acceptance Criteria)

- [ ] **M1:** 커밋 시 **변경 구획만** 요약이 (재)생성, 미변경 구획은 재계산 안 됨(비용 계약테스트). 요약에 기준 커밋·구획 Critic 헬스 스탬프.
- [ ] **M2:** `api/rag/entrypoint`가 진입노드에서 멈추지 않고 **이웃·경로 확장 후 답변 합성**까지 수행, 구획 스코프 안에서 답하고 근거 칩 부착. 진단 질의가 원인후보·조치추천을 근거와 함께 제시(확정은 HITL).
- [ ] **M3:** 전역 질의가 단일 구획 top-k가 아니라 **다수 구획 요약 map-reduce**로 답하고 근거 구획 칩 부착.
- [ ] **M4:** 자연어 목표에 대해 추론 체인의 각 단계 근거(구획·노드·경로·함수·커밋)가 노출됨. Assist는 읽기·탐색 보조에 한정, 변경은 기존 흐름 우회.
- [ ] **M5:** 에이전트가 이상감지→원인분석→**proposal**을 액션보드(PRD-G)에 올리고, 실행 확정 없이는 그래프가 변경되지 않음. 제안에 근거경로·신뢰도 포함.
- [ ] **가드레일:** 전 표면 그래프 1비트도 변경 안 함(읽기전용 계약테스트). 근거 없는 문장은 "모델에 근거 없음". 자율실행 0건.
- [ ] **공통:** 신규 UI shadcn/ui·한국어·기존 배지/카드 taxonomy·시각언어(원/점/파선/실선) 준수, lint·프로덕션 빌드·기존 테스트 회귀 0.

## 7. 결론 (비판적 옹호 요약)

팔란티어 AIP(Logic/Assist/Agents)는 "온톨로지 위에서 답하고·추론하고·조치를 제안하는 표면"에서 돈을 벌지만, 그 표면조차 **자율실행이 아니라 proposal+사람 검토**다 — 즉 AIP의 성공은 우리 북극성(AI=Critic)의 정당성을 입증한다. 우리는 AIP가 무거운 파이프라인·코드 함수로 얻는 것을 **이미 가진 자산의 재배치**로 얻는다: 구획=커뮤니티, provenance=인용, Critic=신뢰, 완성된 Text2Cypher 읽기전용 가드, pgvector, 그리고 절반만 완성된 `api/rag/entrypoint`. 이 PRD는 그 스캐폴드를 **국소·진단·전역·추론·제안**의 완성 표면으로 잇되, 정직 경계(완전자동 금지, 근거 없으면 침묵, 소비는 읽기전용)를 불변식으로 못박는다. 차별점은 한 문장이다 — **"경쟁 GraphRAG는 노이즈 그래프에 답을 얹지만, 우리는 사람이 검수한 감사 가능한 모델 위에서 근거경로로 인용해 답하고, 에이전트는 실행이 아니라 제안한다."** 북극성 "운영 온톨로지"에서 '운영'이 마침내 사용자의 질문에 답하고, 조치를 제안하는 순간.

## 8. 열린 결정 / 불가 기능 (Open Decisions / Out of Scope)

### 열린 결정 (Open Decisions)
1. **DRIFT 라우팅 기본값:** 애매 질의를 국소·전역 둘 다 실행 후 병합(정확·고비용) vs 소형 LLM 분류 후 단일 경로(저비용·오분류 위험) — 기본 정책 결정 필요.
2. **패키징 경계:** 무료=국소 질의, 유료=전역 사센스메이킹+진단 RAG+제안형 에이전트 — 어디에 유료 선을 그을지(에이전트를 최상위 티어로 격리할지).
3. **에이전트 이상감지 트리거:** 사용자 수동 실행 vs 스케줄·이벤트 기반 자동 감지(읽기전용이라도 "자동 감지"가 자율실행 오해를 줄지) — 정직 메시징과 충돌 검토 필요.
4. **요약 롤업 깊이:** is-a 계층 전 층위 요약 vs 상위 N층만(신선도·비용 트레이드오프).
5. **추론 체인 표시 상세도(M4):** 전 단계 근거를 항상 노출 vs 접힘 기본(정보 과부하 vs 투명성) — UX 결정.
6. **구획 없는(단일 구획) 모델의 전역 검색:** is-a 계층을 커뮤니티 대용으로 쓸지, 전역 검색을 비활성할지.

### 불가 기능 (Out of Scope — 명시적 Out)
1. **에이전트 자율실행** — 감지→분석→**제안까지**. 실행 확정은 반드시 사람(HITL). 금지.
2. **소비 표면의 모델 변경(writeback)** — 소비·에이전트 전 표면 읽기전용. 모델 변경은 별도 생성흐름/PRD-G 액션보드로만.
3. **GraphRAG 인덱서 재구현** — Leiden 군집화·엔티티 자동 추출·전량 커뮤니티 요약 파이프라인 신축 안 함(구획·계층 재사용).
4. **AIP Logic식 TypeScript 함수 저작 IDE** — 결정함수 저작은 PRD-B(자연어+HITL 컨펌). 이 PRD는 함수를 **읽어 추론에 엮는 것**까지.
5. **라이브 데이터 파이프라인·실시간 writeback 제어 루프** — 팔란티어형 sense→act 자동 루프는 범위 밖(전략노트 kinetic 논의 참조, 별도 PRD).
6. **근거 없는 답변 생성** — 인용 못 붙는 문장은 답 대신 "모델에 근거 없음". 환각 허용 안 함.
